C# vs Python

Contexto

¿Por qué compararlos?

C# y Python representan dos filosofías distintas de desarrollo. Conocer ambos amplía tu perspectiva como desarrollador.

C# — Tipado estático

Compilado, fuertemente tipado, orientado a objetos. Domina en desarrollo de escritorio (WPF), gaming (Unity), APIs (.NET) y aplicaciones empresariales.

Python — Tipado dinámico

Interpretado, tipado dinámico, multiparadigma. Domina en data science, IA/ML, automatización, scripting y educación por su sintaxis limpia.

01 · Fundamentos

Estructura básica de un programa

El clásico "Hello World" y la estructura mínima de cada lenguaje.

C# Program.cs
// C# — Sintaxis completa
using System;

namespace MiApp
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hola Mundo");
        }
    }
}

// C# 12 — Top-level statements
Console.WriteLine("Hola Mundo");
Py main.py
# Python — Así de simple
print("Hola Mundo")



# Con función main (buena práctica)
def main():
    print("Hola Mundo")

if __name__ == "__main__":
    main()
Nota: Desde C# 9, los top-level statements permiten escribir sin namespace/class, acercándose a la simplicidad de Python.

03 · Control de flujo

Condicionales

C# if / switch
// If-else
if (edad >= 18)
{
    Console.WriteLine("Mayor");
}
else if (edad >= 13)
{
    Console.WriteLine("Adolescente");
}
else
{
    Console.WriteLine("Niño");
}

// Ternario
var estado = edad >= 18 ? "Mayor" : "Menor";

// Switch expression (C# 8+)
var msg = edad switch
{
    >= 18 => "Adulto",
    >= 13 => "Teen",
    _     => "Niño"
};
Py if / match
# If-elif-else
if edad >= 18:
    print("Mayor")
elif edad >= 13:
    print("Adolescente")
else:
    print("Niño")



# Ternario
estado = "Mayor" if edad >= 18 else "Menor"

# Match-case (Python 3.10+)
match edad:
    case n if n >= 18:
        msg = "Adulto"
    case n if n >= 13:
        msg = "Teen"
    case _:
        msg = "Niño"
04 · Bucles

Iteración: for, while, foreach

C# Bucles
// For clásico
for (int i = 0; i < 5; i++)
{
    Console.WriteLine(i);
}

// Foreach
var frutas = new[] { "🍎", "🍌", "🍇" };
foreach (var f in frutas)
{
    Console.WriteLine(f);
}

// While
int n = 0;
while (n < 3)
{
    Console.WriteLine(n++);
}
Py Bucles
# For con range
for i in range(5):
    print(i)


# For sobre colección
frutas = ["🍎", "🍌", "🍇"]
for f in frutas:
    print(f)


# While
n = 0
while n < 3:
    print(n)
    n += 1
Diferencia clave: C# usa foreach...in para iterar colecciones; Python simplemente usa for...in para todo. Python no tiene un for clásico con i++.
05 · Funciones

Funciones y métodos

C# Métodos
// Método con tipo de retorno
static double Sumar(double a, double b)
{
    return a + b;
}

// Método void
static void Saludar(string nombre)
{
    Console.WriteLine($"Hola {nombre}");
}

// Parámetro opcional
static void Log(string msg, int nivel = 1)
{
    Console.WriteLine($"[{nivel}] {msg}");
}

// Expression body (arrow)
static int Doble(int x) => x * 2;
Py Funciones
# Función con retorno
def sumar(a: float, b: float) -> float:
    return a + b


# Función sin retorno explícito
def saludar(nombre: str):
    print(f"Hola {nombre}")


# Parámetro opcional
def log(msg: str, nivel: int = 1):
    print(f"[{nivel}] {msg}")


# Lambda
doble = lambda x: x * 2
Equivalencia: Las expression-bodied members de C# (=>) son análogos a las lambda de Python, pero en C# se usan dentro de clases como métodos completos.
06 · OOP

Clases y objetos

C# Clases
public class Persona
{
    // Propiedades
    public string Nombre { get; set; }
    public int Edad { get; set; }

    // Constructor
    public Persona(string nombre, int edad)
    {
        Nombre = nombre;
        Edad = edad;
    }

    // Método
    public string Presentarse()
        => $"Soy {Nombre}, {Edad} años";
}

// Uso
var p = new Persona("Ana", 25);
Console.WriteLine(p.Presentarse());
Py Clases
class Persona:
    # Constructor
    def __init__(self, nombre: str, edad: int):
        self.nombre = nombre
        self.edad = edad



    # Método
    def presentarse(self) -> str:
        return f"Soy {self.nombre}, {self.edad} años"



# Uso
p = Persona("Ana", 25)
print(p.presentarse())
Diferencia clave: C# usa propiedades con getters/setters automáticos. Python usa atributos directos + @property si necesitas control. Python siempre pasa self explícitamente.
07 · Herencia

Herencia e interfaces

C# Herencia
// Interfaz
public interface IMovible
{
    void Mover(double dx, double dy);
}

// Clase base
public class Forma
{
    public virtual double Area()
        => 0;
}

// Herencia + interfaz
public class Circulo : Forma, IMovible
{
    public double Radio { get; set; }

    public override double Area()
        => Math.PI * Radio * Radio;

    public void Mover(double dx, double dy)
        => Console.WriteLine($"Moviendo {dx},{dy}");
}
Py Herencia
from abc import ABC, abstractmethod
import math

# Clase abstracta (interfaz)
class Movible(ABC):
    @abstractmethod
    def mover(self, dx: float, dy: float): ...

# Clase base
class Forma:
    def area(self) -> float:
        return 0

# Herencia múltiple
class Circulo(Forma, Movible):
    def __init__(self, radio: float):
        self.radio = radio

    def area(self) -> float:
        return math.pi * self.radio ** 2

    def mover(self, dx: float, dy: float):
        print(f"Moviendo {dx},{dy}")

Py Colecciones
# Lista (mutable)
nums = [1, 2, 3, 4, 5]

# Lista
lista = ["a", "b", "c"]
lista.append("d")

# Diccionario

09 · Queries

LINQ vs List Comprehensions

Uno de los puntos más poderosos de cada lenguaje: transformar y consultar datos.

C# LINQ
var numeros = new[] { 1, 2, 3, 4, 5, 6, 7, 8 };

// Filtrar pares
var pares = numeros
    .Where(n => n % 2 == 0)
    .ToList();

// Transformar
var cuadrados = numeros
    .Select(n => n * n)
    .ToList();

// Encadenar operaciones
var resultado = numeros
    .Where(n => n > 3)
    .Select(n => n * 2)
    .OrderByDescending(n => n)
    .Take(3)
    .ToList();
Py Comprehensions
numeros = [1, 2, 3, 4, 5, 6, 7, 8]

# Filtrar pares
pares = [n for n in numeros
         if n % 2 == 0]

# Transformar
cuadrados = [n * n
             for n in numeros]

# Encadenar operaciones
resultado = sorted(
    [n * 2 for n in numeros if n > 3],
    reverse=True
)[:3]
Equivalencia: .Where() ≈ if en comprehension | .Select() ≈ expresión del comprehension | .OrderBy() ≈ sorted() | .Take(n) ≈ slicing [:n]
10 · Errores

Manejo de excepciones

C# try-catch
try
{
    var resultado = 10 / 0;
}
catch (DivideByZeroException ex)
{
    Console.WriteLine(
      $"Error: {ex.Message}");
}
catch (Exception ex)
{
    Console.WriteLine(
      $"General: {ex.Message}");
}
finally
{
    Console.WriteLine("Fin");
}

// Lanzar excepción
throw new ArgumentException("Inválido");
Py try-except
try:
    resultado = 10 / 0

except ZeroDivisionError as e:
    print(f"Error: {e}")

except Exception as e:
    print(f"General: {e}")

finally:
    print("Fin")




# Lanzar excepción
raise ValueError("Inválido")
Mapeo: catch → except | throw → raise | finally = igual en ambos | C# usa new para crear excepciones, Python no.
11 · Async

Programación asíncrona

C# async/await
using System.Net.Http;

static async Task<string> FetchData(
    string url)
{
    using var client = new HttpClient();
    var response = await client
        .GetStringAsync(url);
    return response;
}

// Llamada
var data = await FetchData(
    "https://api.example.com");

// Múltiples tareas paralelas
var tasks = urls.Select(
    u => FetchData(u));
var results = await Task
    .WhenAll(tasks);
Py async/await
import aiohttp
import asyncio

async def fetch_data(
    url: str) -> str:
    async with aiohttp.ClientSession() as s:
        async with s.get(url) as resp:
            return await resp.text()

# Llamada
data = await fetch_data(
    "https://api.example.com")

# Múltiples tareas paralelas
tasks = [fetch_data(u)
         for u in urls]
results = await asyncio.gather(
    *tasks)
Paralelo: Task.WhenAll() en C# ≈ asyncio.gather() en Python. Ambos retornan cuando todas las tareas terminan. C# usa Task<T>, Python usa coroutines.

Conclusión

Dos lenguajes, una lógica

Los conceptos fundamentales son los mismos. Lo que cambia es la sintaxis y las convenciones. Aprender uno hace más fácil dominar el otro.

C#

Explícito, seguro, enterprise-ready. Ideal para aplicaciones robustas y Revit API.

Ambos

OOP, async/await, excepciones, LINQ ↔ comprehensions. La lógica trasciende.

Python

Conciso, flexible, rápido de prototipar. Ideal para scripting y automatización AEC.
